<span data-ttu-id="be42a-105">C# 언어의 예외 처리 기능은 프로그램이 실행 중일 때 발생하는 예기치 않은 문제나 예외 상황을 처리하는 데 도움이 됩니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-105">The C# language's exception handling features help you deal with any unexpected or exceptional situations that occur when a program is running.</span></span> <span data-ttu-id="be42a-106">예외 처리는 `try`, `catch` 및 `finally` 키워드를 사용하여 실패했을 수 있는 작업을 시도하고, 실패를 처리하는 것이 적절하다고 판단될 때 처리하고, 리소스를 정리합니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-106">Exception handling uses the `try`, `catch`, and `finally` keywords to try actions that may not succeed, to handle failures when you decide that it's reasonable to do so, and to clean up resources afterward.</span></span> <span data-ttu-id="be42a-107">예외는 CLR(공용 언어 런타임), .NET, 타사 라이브러리 또는 애플리케이션 코드에서 생성될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-107">Exceptions can be generated by the common language runtime (CLR), by .NET or third-party libraries, or by application code.</span></span> <span data-ttu-id="be42a-108">예외는 `throw` 키워드를 사용하여 생성됩니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-108">Exceptions are created by using the `throw` keyword.</span></span>

## <a name="exceptions-overview"></a><span data-ttu-id="be42a-114">예외 개요</span><span class="sxs-lookup"><span data-stu-id="be42a-114">Exceptions Overview</span></span>

<span data-ttu-id="be42a-115">예외는 다음과 같은 속성을 갖습니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-115">Exceptions have the following properties:</span></span>

- <span data-ttu-id="be42a-116">모든 예외는 궁극적으로 `System.Exception`에서 파생되는 형식입니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-116">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>
- <span data-ttu-id="be42a-117">예외를 throw할 수 있는 문 주위에 `try` 블록을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-117">Use a `try` block around the statements that might throw exceptions.</span></span>
- <span data-ttu-id="be42a-118">`try` 블록에서 예외가 발생하면 제어 흐름이 호출 스택에 있는 첫 번째 관련 예외 처리기로 이동됩니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-118">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="be42a-119">C#에서 `catch` 키워드는 예외 처리기를 정의하는 데 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-119">In C#, the `catch` keyword is used to define an exception handler.</span></span>
- <span data-ttu-id="be42a-120">지정된 예외에 대한 예외 처리기가 없으면 프로그램은 오류 메시지를 나타내며 실행을 중지합니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-120">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>
- <span data-ttu-id="be42a-121">예외를 처리하고 애플리케이션을 알려진 상태로 둘 수 없으면 예외를 catch하지 마세요.</span><span class="sxs-lookup"><span data-stu-id="be42a-121">Don't catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="be42a-122">`System.Exception`을 catch하는 경우 `catch` 블록 끝에서 `throw` 키워드를 사용하여 다시 throw하세요.</span><span class="sxs-lookup"><span data-stu-id="be42a-122">If you catch `System.Exception`, rethrow it using the `throw` keyword at the end of the `catch` block.</span></span>
- <span data-ttu-id="be42a-123">`catch` 블록이 예외 변수를 정의하는 경우 이 변수를 사용하여 발생한 예외 형식에 대한 추가 정보를 얻을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-123">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>
- <span data-ttu-id="be42a-124">`throw` 키워드를 사용하여 프로그램에서 명시적으로 예외를 생성할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-124">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>
- <span data-ttu-id="be42a-125">예외 개체는 호출 스택의 상태 및 오류에 대한 텍스트 설명 같은 오류에 대한 자세한 정보를 포함합니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-125">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>
- <span data-ttu-id="be42a-126">`finally` 블록의 코드는 예외가 throw되더라도 실행됩니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-126">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="be42a-127">`finally` 블록을 사용하여 `try` 블록에서 열려 있는 스트림이나 파일을 닫는 것처럼 리소스를 해제합니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-127">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>
- <span data-ttu-id="be42a-128">.NET의 관리되는 예외는 Win32 구조적 예외 처리 메커니즘을 토대로 구현됩니다.</span><span class="sxs-lookup"><span data-stu-id="be42a-128">Managed exceptions in .NET are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="be42a-129">자세한 내용은 [구조적 예외 처리(C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) 및 [Win32 구조적 예외 처리에 대한 집중 과정](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm)을 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="be42a-129">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span></span>
