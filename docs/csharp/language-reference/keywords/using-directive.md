## <a name="using-static-type"></a><span data-ttu-id="f2321-112">정적 형식 사용</span><span class="sxs-lookup"><span data-stu-id="f2321-112">Using static type</span></span>

<span data-ttu-id="f2321-113">형식 이름을 사용하여 액세스를 한정할 필요 없이 형식의 정적 멤버에 액세스할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-113">You can access static members of a type without having to qualify the access with the type name:</span></span>

```csharp
    static void Main()
    {
        WriteLine(Sqrt(3*3 + 4*4));
    }
}
```

<span data-ttu-id="f2321-115">`using` 지시문의 범위는 지시문이 나타내는 파일로 제한됩니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-115">The scope of a `using` directive is limited to the file in which it appears.</span></span>

<span data-ttu-id="f2321-120">`using` 별칭 지시문을 만들면 네임스페이스 또는 형식에 대한 식별자를 더 쉽게 한정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-120">Create a `using` alias directive to make it easier to qualify an identifier to a namespace or type.</span></span> <span data-ttu-id="f2321-121">모든 `using` 지시문에서 앞에 오는 `using` 지시문에 관계없이 정규화된 네임스페이스 또는 형식을 사용해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-121">In any `using` directive, the fully-qualified namespace or type must be used regardless of the `using` directives that come before it.</span></span> <span data-ttu-id="f2321-122">`using` 별칭은 `using` 지시문 선언에 사용할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-122">No `using` alias can be used in the declaration of a `using` directive.</span></span> <span data-ttu-id="f2321-123">예를 들어, 다음은 컴파일러 오류를 생성합니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-123">For example, the following generates a compiler error:</span></span>

<span data-ttu-id="f2321-124">`using` 지시문을 만들어서 네임스페이스를 지정할 필요 없이 네임스페이스에서 이 형식을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-124">Create a `using` directive to use the types in a namespace without having to specify the namespace.</span></span> <span data-ttu-id="f2321-125">`using` 지시문은 지정한 네임스페이스에 중첩된 모든 네임스페이스에 대한 액세스 권한을 제공하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-125">A `using` directive does not give you access to any namespaces that are nested in the namespace you specify.</span></span>

<span data-ttu-id="f2321-126">네임스페이스는 두 가지 범주인 사용자 정의 및 시스템 정의로 구분됩니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-126">Namespaces come in two categories: user-defined and system-defined.</span></span> <span data-ttu-id="f2321-127">사용자 정의 네임스페이스는 코드에서 정의된 네임스페이스입니다.</span><span class="sxs-lookup"><span data-stu-id="f2321-127">User-defined namespaces are namespaces defined in your code.</span></span> <span data-ttu-id="f2321-128">시스템 정의 네임스페이스 목록은 [.NET API 브라우저](../../../../api/index.md)를 참조하세요.</span><span class="sxs-lookup"><span data-stu-id="f2321-128">For a list of the system-defined namespaces, see [.NET API Browser](../../../../api/index.md).</span></span>
