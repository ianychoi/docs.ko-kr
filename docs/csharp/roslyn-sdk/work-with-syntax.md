## <a name="syntax-trees"></a><span data-ttu-id="8880c-110">구문 트리</span><span class="sxs-lookup"><span data-stu-id="8880c-110">Syntax trees</span></span>

<span data-ttu-id="8880c-111">구문 트리는 컴파일, 코드 분석, 바인딩, 리팩터링, IDE 기능 및 코드 생성에 사용되는 기본 구조입니다.</span><span class="sxs-lookup"><span data-stu-id="8880c-111">Syntax trees are the primary structure used for compilation, code analysis, binding, refactoring, IDE features, and code generation.</span></span> <span data-ttu-id="8880c-112">소스 코드의 어떠한 부분도 먼저 식별되고 잘 알려진 많은 구조적 언어 요소 중 하나로 분류되지 않고 인식되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="8880c-112">No part of the source code is understood without it first being identified and categorized into one of many well-known structural language elements.</span></span>

<span data-ttu-id="8880c-148">구문 토큰은 코드의 가장 작은 구문 조각을 나타내는 언어 문법의 터미널입니다.</span><span class="sxs-lookup"><span data-stu-id="8880c-148">Syntax tokens are the terminals of the language grammar, representing the smallest syntactic fragments of the code.</span></span> <span data-ttu-id="8880c-149">다른 노드 또는 토큰의 부모가 아닙니다.</span><span class="sxs-lookup"><span data-stu-id="8880c-149">They are never parents of other nodes or tokens.</span></span> <span data-ttu-id="8880c-150">구문 토큰은 키워드, 식별자, 리터럴 및 문장 부호로 구성됩니다.</span><span class="sxs-lookup"><span data-stu-id="8880c-150">Syntax tokens consist of keywords, identifiers, literals, and punctuation.</span></span>
