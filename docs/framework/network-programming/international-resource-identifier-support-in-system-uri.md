# <a name="international-resource-identifier-support-in-systemuri"></a><span data-ttu-id="954f0-102">System.Uri의 국가별 리소스 식별자 지원</span><span class="sxs-lookup"><span data-stu-id="954f0-102">International Resource Identifier Support in System.Uri</span></span>

<span data-ttu-id="954f0-103"><xref:System.Uri?displayProperty=nameWithType> 클래스는 IRI(International Resource Identifier) 및 IDN(Internationalized Domain Names)을 통해 확장되었습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-103">The <xref:System.Uri?displayProperty=nameWithType> class has been extended with International Resource Identifier (IRI) and Internationalized Domain Names (IDN) support.</span></span> <span data-ttu-id="954f0-104">이러한 향상된 기능은 .NET Framework 3.5, 3.0 SP1 및 2.0 SP1에서 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-104">These enhancements are available in .NET Framework 3.5, 3.0 SP1, and 2.0 SP1.</span></span>  
  
## <a name="iri-and-idn-support"></a><span data-ttu-id="954f0-105">IRI 및 IDN 지원</span><span class="sxs-lookup"><span data-stu-id="954f0-105">IRI and IDN Support</span></span>  

 <span data-ttu-id="954f0-106">웹 주소는 대개 매우 제한적인 문자 집합으로 구성된 URI(Uniform Resource Identifier)를 사용하여 표현됩니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-106">Web addresses are typically expressed using Uniform Resource Identifiers (URI) that consist of a very restricted set of characters:</span></span>  
  
- <span data-ttu-id="954f0-107">영문자의 대문자 및 소문자 ASCII 문자.</span><span class="sxs-lookup"><span data-stu-id="954f0-107">Upper and lower case ASCII letters from the English alphabet.</span></span>  
  
- <span data-ttu-id="954f0-108">0~9의 숫자.</span><span class="sxs-lookup"><span data-stu-id="954f0-108">Digits from 0 to 9.</span></span>  
  
- <span data-ttu-id="954f0-109">일부 기타 ASCII 기호.</span><span class="sxs-lookup"><span data-stu-id="954f0-109">A small number of other ASCII symbols.</span></span>  
  
 <span data-ttu-id="954f0-110">URI 사양은 IETF(Internet Engineering Task Force)에서 게시한 RFC 2396 및 RFC 3986에 설명되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-110">The specifications for URIs are documented in RFC 2396 and RFC 3986 published by the Internet Engineering Task Force (IETF).</span></span>  
  
 <span data-ttu-id="954f0-111">인터넷이 성장하면서 영어 이외의 언어로 리소스를 식별해야 할 필요성이 증가하고 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-111">With the growth of the Internet, there is a growing need to identify resources using languages other than English.</span></span> <span data-ttu-id="954f0-112">이 필요성을 가능하게 하고 ASCII 이외 문자(유니코드/ISO 10646 문자 집합의 문자)를 허용하는 식별자를 IRI(International Resource Identifier)라고 합니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-112">Identifiers which facilitate this need and allow non-ASCII characters (characters in the Unicode/ISO 10646 character set) are known as International Resource Identifiers (IRIs).</span></span> <span data-ttu-id="954f0-113">IRI 사양은 IETF에서 게시한 RFC 3987에 설명되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-113">The specifications for IRIs are documented in RFC 3987 published by IETF.</span></span> <span data-ttu-id="954f0-114">IRI를 사용하면 URL에 유니코드 문자를 포함할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-114">Using IRIs allows a URL to contain Unicode characters.</span></span>  
  
 <span data-ttu-id="954f0-115">기존 <xref:System.Uri?displayProperty=nameWithType> 클래스는 RFC 3987을 기반으로 IRI 지원을 제공하도록 확장되었습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-115">The existing <xref:System.Uri?displayProperty=nameWithType> class has been extended to provide IRI support based on RFC 3987.</span></span> <span data-ttu-id="954f0-116">현재 사용자의 경우 IRI를 사용하도록 설정하지 않는 한 .NET Framework 2.0 동작에서 차이를 느끼지 못할 것입니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-116">Current users will not see any change from the .NET Framework 2.0 behavior unless they specifically enable IRI.</span></span> <span data-ttu-id="954f0-117">이 덕분에 .NET Framework 이전 버전과의 애플리케이션 호환성이 제공됩니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-117">This ensures application compatibility with prior versions of the .NET Framework.</span></span>  
  
 <span data-ttu-id="954f0-118">애플리케이션은 도메인 이름에 적용되는 IDN(다국어 도메인 이름) 구문 분석을 사용할지 여부와 IRI 구문 분석 규칙을 적용할지 여부를 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-118">An application can specify whether to use Internationalized Domain Name (IDN) parsing applied to domain names and whether IRI parsing rules should be applied.</span></span> <span data-ttu-id="954f0-119">이 설정은 machine.config 또는 app.config 파일에서 지정할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-119">This can be done in the machine.config or in the app.config file.</span></span>  
  
 <span data-ttu-id="954f0-120">IDN을 사용하면 도메인 이름의 모든 유니코드 레이블이 해당 Punycode 문자로 변환됩니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-120">Enabling IDN will convert all Unicode labels in a domain name to their Punycode equivalents.</span></span> <span data-ttu-id="954f0-121">Punycode 이름에는 ASCII 문자만 사용되며 항상 xn-- 접두사로 시작합니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-121">Punycode names contain only ASCII characters and always start with the xn-- prefix.</span></span> <span data-ttu-id="954f0-122">대부분의 DNS 서버는 ASCII 문자만 지원하므로(RFC 3940 참조) 이렇게 해야 인터넷에서 기존 DNS 서버를 지원할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-122">The reason for this is to support existing DNS servers on the Internet, since most DNS servers only support ASCII characters (see RFC 3940).</span></span>  
  
 <span data-ttu-id="954f0-123">IRI 및 IDN을 사용하도록 설정하면 <xref:System.Uri.DnsSafeHost%2A?displayProperty=nameWithType> 속성 값에 영향을 줍니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-123">Enabling IRI and IDN affects the value of the <xref:System.Uri.DnsSafeHost%2A?displayProperty=nameWithType> property.</span></span> <span data-ttu-id="954f0-124">IRI 및 IDN을 사용하면 <xref:System.Uri.Equals%2A?displayProperty=nameWithType>, <xref:System.Uri.OriginalString%2A?displayProperty=nameWithType>, <xref:System.Uri.GetComponents%2A?displayProperty=nameWithType> 및 <xref:System.Uri.IsWellFormedOriginalString%2A> 메서드의 동작을 변경할 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-124">Enabling IRI and IDN can also change the behavior of the <xref:System.Uri.Equals%2A?displayProperty=nameWithType>, <xref:System.Uri.OriginalString%2A?displayProperty=nameWithType>, <xref:System.Uri.GetComponents%2A?displayProperty=nameWithType>, and <xref:System.Uri.IsWellFormedOriginalString%2A> methods.</span></span>  
  
 <span data-ttu-id="954f0-125">또한 <xref:System.GenericUriParser?displayProperty=nameWithType> 클래스는 IRI 및 IDN을 지원하는 사용자 지정 가능한 파서를 만들 수 있도록 확장되었습니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-125">The <xref:System.GenericUriParser?displayProperty=nameWithType> class has also been extended to allow creating a customizable parser that supports IRI and IDN.</span></span> <span data-ttu-id="954f0-126"><xref:System.GenericUriParser?displayProperty=nameWithType> 개체의 동작은 <xref:System.GenericUriParserOptions?displayProperty=nameWithType> 열거형에서 사용 가능한 값의 비트 조합을 <xref:System.GenericUriParser?displayProperty=nameWithType> 생성자에 전달하여 지정합니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-126">The behavior of a <xref:System.GenericUriParser?displayProperty=nameWithType> object is specified by passing a bitwise combination of the values available in the <xref:System.GenericUriParserOptions?displayProperty=nameWithType> enumeration to the <xref:System.GenericUriParser?displayProperty=nameWithType> constructor.</span></span> <span data-ttu-id="954f0-127"><xref:System.GenericUriParserOptions.IriParsing?displayProperty=nameWithType> 형식은 파서가 RFC 3987에 지정된 IRI(International Resource Identifier)에 대한 구문 분석 규칙을 지원함을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-127">The <xref:System.GenericUriParserOptions.IriParsing?displayProperty=nameWithType> type indicates the parser supports the parsing rules specified in RFC 3987 for International Resource Identifiers (IRI).</span></span> <span data-ttu-id="954f0-128">IRI가 실제로 사용되는지 여부는 IRI가 사용하도록 설정되었는지에 따라 다릅니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-128">Whether IRI is actually used depends on if IRI is enabled.</span></span>  
  
 <span data-ttu-id="954f0-129"><xref:System.GenericUriParserOptions.Idn?displayProperty=nameWithType> 형식은 파서가 호스트 이름의 IDN(Internationalized Domain Name) 구문 분석을 지원함을 나타냅니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-129">The <xref:System.GenericUriParserOptions.Idn?displayProperty=nameWithType> type indicates the parser supports Internationalized Domain Name (IDN) parsing (IDN) of host names.</span></span> <span data-ttu-id="954f0-130">IDN이 실제로 사용되는지 여부는 IDN이 사용하도록 설정되었는지에 따라 다릅니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-130">Whether IDN is actually used depends on if IDN is enabled.</span></span>  
  
 <span data-ttu-id="954f0-131">IRI 구문 분석을 사용하도록 설명하면 RFC 3987의 최신 IRI 규칙에 따라 정규화 및 문자 검사가 수행됩니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-131">Enabling IRI parsing will do normalization and character checking according to the latest IRI rules in RFC 3987.</span></span> <span data-ttu-id="954f0-132">IRI 구문 분석의 기본값은 사용하지 않도록 설정되는 것이므로 정규화 및 문자 검사는 RFC 2396 및 RFC 3986에 따라 수행됩니다.</span><span class="sxs-lookup"><span data-stu-id="954f0-132">The default value is for IRI parsing to be disabled so normalization and character checking are done according to RFC 2396 and RFC 3986.</span></span>
