## <a name="wcf-security-functional-areas"></a><span data-ttu-id="2bb41-170">WCF 보안 기능 영역</span><span class="sxs-lookup"><span data-stu-id="2bb41-170">WCF Security Functional Areas</span></span>  

 <span data-ttu-id="2bb41-171">WCF 보안은 전송 보안, 액세스 제어 및 감사의 세 가지 기능 영역으로 구분 됩니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-171">WCF security is divided into three functional areas: transfer security, access control, and auditing.</span></span> <span data-ttu-id="2bb41-172">다음 단원에서는 이러한 영역에 대해 간략하게 설명하고 추가 정보를 위한 링크를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-172">The following sections briefly discuss these areas and provide links for more information.</span></span>  
  
### <a name="transfer-security"></a><span data-ttu-id="2bb41-173">전송 보안</span><span class="sxs-lookup"><span data-stu-id="2bb41-173">Transfer Security</span></span>  

 <span data-ttu-id="2bb41-174">전송 보안은 무결성, 기밀성 및 인증의 세 가지 주요 보안 기능을 포함합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-174">Transfer security encompasses three major security functions: integrity, confidentiality, and authentication.</span></span> <span data-ttu-id="2bb41-175">*무결성* 은 메시지가 변조 되었는지 여부를 검색할 수 있는 기능입니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-175">*Integrity* is the ability to detect whether a message has been tampered with.</span></span> <span data-ttu-id="2bb41-176">*기밀성* 은 의도 된 받는 사람이 아닌 다른 사람이 메시지를 읽을 수 없도록 하는 기능입니다. 이는 암호화를 통해 달성 됩니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-176">*Confidentiality* is the ability to keep a message unreadable by anyone other than the intended recipient; this is achieved through cryptography.</span></span> <span data-ttu-id="2bb41-177">*인증은* 요청 된 id를 확인할 수 있는 기능입니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-177">*Authentication* is the ability to verify a claimed identity.</span></span> <span data-ttu-id="2bb41-178">이러한 세 기능을 함께 사용하면 메시지를 지점 간에 안전하게 전달할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-178">Together, these three functions help to ensure that messages securely arrive from one point to another.</span></span>  
  
#### <a name="transport-and-message-security-modes"></a><span data-ttu-id="2bb41-179">전송 및 메시지 보안 모드</span><span class="sxs-lookup"><span data-stu-id="2bb41-179">Transport and Message Security Modes</span></span>  

 <span data-ttu-id="2bb41-180">WCF에서 전송 보안을 구현 하는 데는 *전송* 보안 모드와 *메시지* 보안 모드의 두 가지 기본 메커니즘이 사용 됩니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-180">Two main mechanisms are used to implement transfer security in WCF: *transport* security mode and *message* security mode.</span></span>  
  
- <span data-ttu-id="2bb41-181">*전송 보안 모드* 에서는 HTTPS와 같은 전송 수준 프로토콜을 사용 하 여 전송 보안을 구현 합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-181">*Transport security mode* uses a transport-level protocol, such as HTTPS, to achieve transfer security.</span></span> <span data-ttu-id="2bb41-182">전송 모드는 광범위하게 채택되고, 다양한 플랫폼에서 사용 가능하며, 계산 과정이 복잡하지 않다는 이점이 있는 한편</span><span class="sxs-lookup"><span data-stu-id="2bb41-182">Transport mode has the advantage of being widely adopted, available on many platforms, and less computationally complex.</span></span> <span data-ttu-id="2bb41-183">지점 간 메시지의 보안만 유지할 수 있다는 단점이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-183">However, it has the disadvantage of securing messages only from point-to-point.</span></span>  
  
- <span data-ttu-id="2bb41-184">반면에 *메시지 보안 모드* 는 WS-Security (및 기타 사양)를 사용 하 여 전송 보안을 구현 합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-184">*Message security mode*, on the other hand, uses WS-Security (and other specifications) to implement transfer security.</span></span> <span data-ttu-id="2bb41-185">메시지 보안은 SOAP 메시지에 직접 적용되고 SOAP 봉투에 애플리케이션 데이터와 함께 포함되기 때문에 전송 프로토콜에 독립적이고, 확장성이 뛰어나며, 엔드투엔드 보안(지점 간과 비교)을 보장한다는 이점이 있습니다. 반면 SOAP 메시지의 XML 특성을 처리해야 하기 때문에 전송 보안 모드보다 몇 배 더 느리다는 단점이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-185">Because the message security is applied directly to the SOAP messages and is contained inside the SOAP envelopes, together with the application data, it has the advantage of being transport protocol-independent, more extensible, and ensuring end-to-end security (versus point-to-point); it has the disadvantage of being several times slower than transport security mode because it has to deal with the XML nature of the SOAP messages.</span></span>  
  
 <span data-ttu-id="2bb41-186">이러한 차이점에 대 한 자세한 내용은 [서비스 및 클라이언트 보안](securing-services-and-clients.md)설정을 참조 하세요.</span><span class="sxs-lookup"><span data-stu-id="2bb41-186">For more information about these differences, see [Securing Services and Clients](securing-services-and-clients.md).</span></span>  
  
 <span data-ttu-id="2bb41-187">세 번째 보안 모드는 위 두 가지 모드를 모두 사용하며 두 모드의 이점을 모두 갖추고 있습니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-187">A third security mode uses both previous modes and brings advantages of both.</span></span> <span data-ttu-id="2bb41-188">이 모드를 `TransportWithMessageCredential`이라고 합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-188">This mode is called `TransportWithMessageCredential`.</span></span> <span data-ttu-id="2bb41-189">이 모드에서는 클라이언트를 인증하는 데 메시지 보안을 사용하고 서버를 인증하고 메시지 기밀성과 무결성을 제공하는 데 전송 보안을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-189">In this mode, message security is used to authenticate the client and transport security is used to authenticate the server and provide message confidentiality and integrity.</span></span> <span data-ttu-id="2bb41-190">이로 인해 `TransportWithMessageCredential` 보안 모드는 전송 보안 모드만큼 빠른 속도를 제공하면서 메시지 보안과 같은 방식으로 클라이언트 인증 확장성을 지원합니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-190">Thanks to this, the `TransportWithMessageCredential` security mode is almost as fast as transport security mode and provides client authentication extensibility in the same way as message security.</span></span> <span data-ttu-id="2bb41-191">반면 메시지 보안 모드와는 달리 완벽한 엔드투엔드 보안은 제공하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="2bb41-191">However, unlike message security mode, it does not provide complete end-to-end security.</span></span>
