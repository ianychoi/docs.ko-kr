## <a name="output"></a><span data-ttu-id="3d3ac-108">출력</span><span class="sxs-lookup"><span data-stu-id="3d3ac-108">Output</span></span>  

 <span data-ttu-id="3d3ac-109">MDA에서 관리되는 컨텍스트 및 관리되지 않는 컨텍스트의 매개 변수 또는 필드 형식과 해당 형식을 포함하는 구조체 또는 메서드를 표시합니다.</span><span class="sxs-lookup"><span data-stu-id="3d3ac-109">The MDA displays the type of the parameter or field in the managed and unmanaged contexts, and the structure or method containing the type.</span></span>  <span data-ttu-id="3d3ac-110">다음은 필드에 대한 출력 예입니다.</span><span class="sxs-lookup"><span data-stu-id="3d3ac-110">The following is an example of the output for a field:</span></span>

## <a name="configuration"></a><span data-ttu-id="3d3ac-111">구성</span><span class="sxs-lookup"><span data-stu-id="3d3ac-111">Configuration</span></span>  

 <span data-ttu-id="3d3ac-112">MDA 구성을 통해 관련된 필드 또는 메서드 이름에 따라 보고된 마샬링 정보를 필터링할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3d3ac-112">The MDA configuration allows you to filter the reported marshaling information based on the involved field or method names.</span></span>  <span data-ttu-id="3d3ac-113">다음 예제에서는 `methodFilter`, `fieldFilter` 및 `match` 요소를 사용하여 필터를 지정하는 방법을 보여 줍니다.</span><span class="sxs-lookup"><span data-stu-id="3d3ac-113">The following example shows the use of the `methodFilter`, `fieldFilter`, and `match` elements to specify filters.</span></span>  <span data-ttu-id="3d3ac-114">특성을 `name` 별표 ()로 설정 \* 하면 모든 항목과 일치 합니다.</span><span class="sxs-lookup"><span data-stu-id="3d3ac-114">Setting the `name` attribute to an asterisk (\*) will match everything.</span></span>  
  
```xml  
<mdaConfig>  
  <assistants>  
    <marshaling>  
      <methodFilter>  
        <match name="Method1"/>  
        <match name="Method2"/>  
      </methodFilter>  
      <fieldFilter>  
        <match name="Field1"/>  
        <match name="Field2"/>  
       </fieldFilter>  
    </marshaling>  
  </assistants>  
</mdaConfig>  
```
