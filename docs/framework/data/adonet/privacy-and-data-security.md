## <a name="securing-string-values-in-memory"></a><span data-ttu-id="c8abd-120">메모리 내 문자열 값 보호</span><span class="sxs-lookup"><span data-stu-id="c8abd-120">Securing String Values in Memory</span></span>  

 <span data-ttu-id="c8abd-121">애플리케이션은 컴퓨터 메모리에서 데이터를 삭제할 수 없으므로 <xref:System.String> 개체에 암호, 신용 카드 번호, 개인 데이터 등의 중요한 정보가 포함되어 있는 경우 해당 데이터가 사용된 후 외부에 노출될 위험이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-121">If a <xref:System.String> object contains sensitive information, such as a password, credit card number, or personal data, there is a risk that the information could be revealed after it is used because the application cannot delete the data from computer memory.</span></span>  
  
 <span data-ttu-id="c8abd-122"><xref:System.String>은 변경할 수 없습니다. 즉, 한번 만들어진 후에는 수정할 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-122">A <xref:System.String> is immutable; its value cannot be modified once it has been created.</span></span> <span data-ttu-id="c8abd-123">이 문자열 값이 수정되어 변경된 것처럼 보일 때도 있습니다. 하지만 실제로는 메모리 내 <xref:System.String> 개체의 새 인스턴스가 만들어지고 데이터가 일반 텍스트로 저장된 것입니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-123">Changes that appear to modify the string value actually create a new instance of a <xref:System.String> object in memory, storing the data as plain text.</span></span> <span data-ttu-id="c8abd-124">뿐만 아니라 이 문자열 인스턴스가 언제 메모리에서 삭제될지 예측할 수도 없습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-124">In addition, it is not possible to predict when the string instances will be deleted from memory.</span></span> <span data-ttu-id="c8abd-125">.NET 가비지 수집의 경우 문자열이 포함된 메모리 회수는 명확하게 수행되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-125">Memory reclamation with strings is not deterministic with .NET garbage collection.</span></span> <span data-ttu-id="c8abd-126">따라서 중요한 데이터에는 <xref:System.String> 및 <xref:System.Text.StringBuilder> 클래스를 사용하지 않아야 합니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-126">You should avoid using the <xref:System.String> and <xref:System.Text.StringBuilder> classes if your data is truly sensitive.</span></span>  
  
 <span data-ttu-id="c8abd-127"><xref:System.Security.SecureString> 클래스는 DPAPI(데이터 보호 API)를 사용하여 메모리 내 텍스트를 암호화하기 위한 메서드를 제공합니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-127">The <xref:System.Security.SecureString> class provides methods for encrypting text using the Data Protection API (DPAPI) in memory.</span></span> <span data-ttu-id="c8abd-128">이 경우 문자열이 더 이상 필요하지 않으면 메모리에서 삭제됩니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-128">The string is then deleted from memory when it is no longer needed.</span></span> <span data-ttu-id="c8abd-129">`ToString`의 내용을 빠르게 읽는 데 사용할 수 있는 <xref:System.Security.SecureString> 메서드는 없습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-129">There is no `ToString` method to quickly read the contents of a <xref:System.Security.SecureString>.</span></span> <span data-ttu-id="c8abd-130">값을 제공하지 않거나 `SecureString` 개체 배열에 대한 포인터를 전달하여 <xref:System.Char>의 새 인스턴스를 초기화할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-130">You can initialize a new instance of `SecureString` with no value or by passing it a pointer to an array of <xref:System.Char> objects.</span></span> <span data-ttu-id="c8abd-131">그런 다음 이 클래스의 다양한 메서드를 사용하여 문자열에 대한 작업을 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="c8abd-131">You can then use the various methods of the class to work with the string.</span></span>
