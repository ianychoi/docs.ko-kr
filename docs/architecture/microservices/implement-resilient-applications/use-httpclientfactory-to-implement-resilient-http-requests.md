<span data-ttu-id="9e8f7-109">따라서 `HttpClient`는 한 번 인스턴스화되어 애플리케이션의 수명 동안 다시 사용됩니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-109">Therefore, `HttpClient` is intended to be instantiated once and reused throughout the life of an application.</span></span> <span data-ttu-id="9e8f7-110">모든 요청에 대해 `HttpClient` 클래스를 인스턴스화하면 과도한 부하에서 사용할 수 있는 소켓 수가 소진됩니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-110">Instantiating an `HttpClient` class for every request will exhaust the number of sockets available under heavy loads.</span></span> <span data-ttu-id="9e8f7-111">이 문제로 인해 `SocketException` 오류가 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-111">That issue will result in `SocketException` errors.</span></span> <span data-ttu-id="9e8f7-112">이 문제를 해결하는 데 가능한 방법은 [HttpClient 사용에 관한 Microsoft 문서](../../../csharp/tutorials/console-webapiclient.md)에서 설명한 대로 `HttpClient` 개체를 singleton 또는 정적으로 만드는 것을 기반으로 합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-112">Possible approaches to solve that problem are based on the creation of the `HttpClient` object as singleton or static, as explained in this [Microsoft article on HttpClient usage](../../../csharp/tutorials/console-webapiclient.md).</span></span> <span data-ttu-id="9e8f7-113">이 방법은 단기 콘솔 앱 또는 하루에 몇 번 실행되는 유사한 앱에 좋은 해결책일 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-113">This can be a good solution for short-lived console apps or similar, that run a few times a day.</span></span>

<span data-ttu-id="9e8f7-114">개발자가 겪는 또 다른 문제는 장기 실행 프로세스에서 `HttpClient`의 공유 인스턴스를 사용하는 경우에 발생합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-114">Another issue that developers run into is when using a shared instance of `HttpClient` in long-running processes.</span></span> <span data-ttu-id="9e8f7-115">HttpClient가 singleton 또는 정적 개체로 인스턴스화되는 상황에서 이 [문제](https://github.com/dotnet/runtime/issues/18348)에서 dotnet/런타임 GitHub 리포지토리에 대해 설명된 대로 DNS 변경 사항을 처리하지 못합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-115">In a situation where the HttpClient is instantiated as a singleton or a static object, it fails to handle the DNS changes as described in this [issue](https://github.com/dotnet/runtime/issues/18348) of the dotnet/runtime GitHub repository.</span></span>

<span data-ttu-id="9e8f7-116">그러나 문제는 `HttpClient` 자체가 아니라 [HttpClient의 기본 생성자](/dotnet/api/system.net.http.httpclient.-ctor?view=netcore-3.1#System_Net_Http_HttpClient__ctor)에 있습니다. 이 생성자는 위에서 언급한 ‘소켓 소모’ 및 DNS 변경 문제가 있는 새로운 구체적인 <xref:System.Net.Http.HttpMessageHandler> 인스턴스를 만들기 때문입니다. </span><span class="sxs-lookup"><span data-stu-id="9e8f7-116">However, the issue isn't really with `HttpClient` per se, but with the [default constructor for HttpClient](/dotnet/api/system.net.http.httpclient.-ctor?view=netcore-3.1#System_Net_Http_HttpClient__ctor), because it creates a new concrete instance of <xref:System.Net.Http.HttpMessageHandler>, which is the one that has *sockets exhaustion* and DNS changes issues mentioned above.</span></span>

<span data-ttu-id="9e8f7-119">[Polly](http://www.thepollyproject.org/)는 미리 정의된 정책을 사용하여 개발자가 흐름 및 스레드로부터 안전한 방식으로 애플리케이션에 복원력을 추가하는 데 도움이 되는 일시적인 오류 처리 라이브러리입니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-119">[Polly](http://www.thepollyproject.org/) is a transient-fault-handling library that helps developers add resiliency to their applications, by using some pre-defined policies in a fluent and thread-safe manner.</span></span>

### <a name="httpclient-lifetimes"></a><span data-ttu-id="9e8f7-160">HttpClient 수명</span><span class="sxs-lookup"><span data-stu-id="9e8f7-160">HttpClient lifetimes</span></span>

<span data-ttu-id="9e8f7-161">`IHttpClientFactory`에서 `HttpClient` 개체를 가져올 때마다 새 인스턴스가 반환됩니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-161">Each time you get an `HttpClient` object from the `IHttpClientFactory`, a new instance is returned.</span></span> <span data-ttu-id="9e8f7-162">그러나 `HttpMessageHandler`의 수명이 만료되지 않은 한, 각 `HttpClient`는 `IHttpClientFactory`에 의해 풀링되어 다시 사용되는 `HttpMessageHandler`를 사용하여 리소스 사용량을 줄입니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-162">But each `HttpClient` uses an `HttpMessageHandler` that's pooled and reused by the `IHttpClientFactory` to reduce resource consumption, as long as the `HttpMessageHandler`'s lifetime hasn't expired.</span></span>

<span data-ttu-id="9e8f7-163">각 처리기가 일반적으로 자체 기본 HTTP 연결을 관리하고 필요 이상으로 처리기를 만드는 것은 연결 지연을 발생시킬 수 있으므로 처리기의 풀링이 바람직합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-163">Pooling of handlers is desirable as each handler typically manages its own underlying HTTP connections; creating more handlers than necessary can result in connection delays.</span></span> <span data-ttu-id="9e8f7-164">또한 일부 처리기는 무한정으로 연결을 열어 놓아 처리기가 DNS 변경에 대응하는 것을 막을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-164">Some handlers also keep connections open indefinitely, which can prevent the handler from reacting to DNS changes.</span></span>

<span data-ttu-id="9e8f7-165">풀의 `HttpMessageHandler` 개체에는 수명이 있으며, 이 수명은 풀의 `HttpMessageHandler` 인스턴스를 다시 사용할 수 있는 기간입니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-165">The `HttpMessageHandler` objects in the pool have a lifetime that's the length of time that an `HttpMessageHandler` instance in the pool can be reused.</span></span> <span data-ttu-id="9e8f7-166">기본값은 2분이지만, 형식화된 클라이언트별로 재정의할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-166">The default value is two minutes, but it can be overridden per Typed Client.</span></span> <span data-ttu-id="9e8f7-167">재정의하려면 다음 코드와 같이 클라이언트를 만들 때 반환되는 <xref:Microsoft.Extensions.DependencyInjection.IHttpClientBuilder>에서 `SetHandlerLifetime()`을 호출합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-167">To override it, call `SetHandlerLifetime()` on the <xref:Microsoft.Extensions.DependencyInjection.IHttpClientBuilder> that's returned when creating the client, as shown in the following code:</span></span>

<span data-ttu-id="9e8f7-168">클라이언트마다 고유하게 구성된 처리기 수명 값이 있을 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-168">Each Typed Client can have its own configured handler lifetime value.</span></span> <span data-ttu-id="9e8f7-169">처리기 만료를 사용하지 않도록 설정하려면 수명을 `InfiniteTimeSpan`으로 설정합니다.</span><span class="sxs-lookup"><span data-stu-id="9e8f7-169">Set the lifetime to `InfiniteTimeSpan` to disable handler expiry.</span></span>
