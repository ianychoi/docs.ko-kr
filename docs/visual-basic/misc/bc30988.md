<span data-ttu-id="a062e-104">클래스의 인스턴스를 선언하는 경우 `New` 키워드 다음에 형식 이름 또는 `With`가 나와야 합니다.</span><span class="sxs-lookup"><span data-stu-id="a062e-104">When you declare an instance of a class, the `New` keyword must be followed by a type name or by `With`.</span></span> <span data-ttu-id="a062e-105">예를 들어 다음 문은 각각 `client` 클래스의 인스턴스가 될 `Customer` 을 선언합니다.</span><span class="sxs-lookup"><span data-stu-id="a062e-105">For example, the following statements each declare `client` to be an instance of the `Customer` class.</span></span> <span data-ttu-id="a062e-106">형식 이름 `Customer` 가 `New`뒤에 나옵니다.</span><span class="sxs-lookup"><span data-stu-id="a062e-106">The type name `Customer` follows `New`.</span></span>  
  
```vb  
' Dim client As New Customer()  
' The next declaration uses an object initializer.  
Dim client As New Customer() With {.Name = "Litware, Inc."}  
```  
  
 <span data-ttu-id="a062e-107">Visual Basic 2008부터 개체를 무명 형식의 인스턴스로 선언할 수 있으며,이 경우 데이터 형식을 지정 하지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="a062e-107">Beginning with Visual Basic 2008, you can declare an object to be an instance of an anonymous type, in which case you do not specify a data type.</span></span> <span data-ttu-id="a062e-108">무명 형식 선언에서 `With` 키워드는 `New`뒤에 나옵니다.</span><span class="sxs-lookup"><span data-stu-id="a062e-108">In anonymous type declarations, the keyword `With` follows `New`.</span></span>  
  
```vb

```
