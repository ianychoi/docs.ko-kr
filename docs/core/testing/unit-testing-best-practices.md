<span data-ttu-id="f536a-302">이러한 문제를 해결하려면 프로덕션 코드에 *이음새* 를 도입해야 합니다.</span><span class="sxs-lookup"><span data-stu-id="f536a-302">To solve these problems, you'll need to introduce a *seam* into your production code.</span></span> <span data-ttu-id="f536a-303">한 가지 방법은 인터페이스에서 제어해야 하는 코드를 래핑하여 프로덕션 코드가 해당 인터페이스에 종속되도록 하는 것입니다.</span><span class="sxs-lookup"><span data-stu-id="f536a-303">One approach is to wrap the code that you need to control in an interface and have the production code depend on that interface.</span></span>

```csharp
public interface IDateTimeProvider
{
    DayOfWeek DayOfWeek();
}

public int GetDiscountedPrice(int price, IDateTimeProvider dateTimeProvider)
{
    if (dateTimeProvider.DayOfWeek() == DayOfWeek.Tuesday)
    {
        return price / 2;
    }
    else
    {
        return price;
    }
}
```

<span data-ttu-id="f536a-304">이제 테스트 도구 모음이 됩니다.</span><span class="sxs-lookup"><span data-stu-id="f536a-304">Your test suite now becomes</span></span>

public void GetDiscountedPrice_NotTuesday_ReturnsFullPrice()

    var priceCalculator = new PriceCalculator();

public void GetDiscountedPrice_OnTuesday_ReturnsHalfPrice()

    var priceCalculator = new PriceCalculator();
    var dateTimeProviderStub = new Mock<IDateTimeProvider>();

    Assert.Equals(1, actual);
}
