## <a name="why-write-async-code"></a><span data-ttu-id="dba8e-113">비동기 코드를 작성하는 이유</span><span class="sxs-lookup"><span data-stu-id="dba8e-113">Why Write Async Code?</span></span>

<span data-ttu-id="dba8e-114">최신 앱은 파일 및 네트워킹 I/O를 광범위하게 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="dba8e-114">Modern apps make extensive use of file and networking I/O.</span></span> <span data-ttu-id="dba8e-115">이전에는 I/O API가 기본적으로 차단되어, 어려운 패턴을 배우고 사용하지 않을 경우 사용자 환경 및 하드웨어 사용률이 좋지 않았습니다.</span><span class="sxs-lookup"><span data-stu-id="dba8e-115">I/O APIs traditionally block by default, resulting in poor user experiences and hardware utilization unless you want to learn and use challenging patterns.</span></span> <span data-ttu-id="dba8e-116">작업 기반 비동기 API 및 언어 수준의 비동기 프로그래밍 모델은 이 모델을 뒤집어 새로운 개념을 배우지 않고도 기본적으로 비동기 실행을 사용할 수 있게 합니다.</span><span class="sxs-lookup"><span data-stu-id="dba8e-116">Task-based async APIs and the language-level asynchronous programming model invert this model, making async execution the default with few new concepts to learn.</span></span>

<span data-ttu-id="dba8e-117">비동기 코드에는 다음과 같은 특성이 있습니다.</span><span class="sxs-lookup"><span data-stu-id="dba8e-117">Async code has the following characteristics:</span></span>

- <span data-ttu-id="dba8e-118">I/O 요청이 반환되기를 기다리는 동안 추가 요청을 처리할 수 있도록 스레드를 양도하여 더 많은 서버 요청을 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="dba8e-118">Handles more server requests by yielding threads to handle more requests while waiting for I/O requests to return.</span></span>
- <span data-ttu-id="dba8e-119">I/O 요청을 기다리는 동안 UI 조작에 스레드를 양도하고 장기 실행 작업을 다른 CPU 코어로 전환하여 UI 응답성을 개선합니다.</span><span class="sxs-lookup"><span data-stu-id="dba8e-119">Enables UIs to be more responsive by yielding threads to UI interaction while waiting for I/O requests and by transitioning long-running work to other CPU cores.</span></span>
