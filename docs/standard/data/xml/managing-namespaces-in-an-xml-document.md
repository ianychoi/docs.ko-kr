# <a name="managing-namespaces-in-an-xml-document"></a><span data-ttu-id="09cec-104">XML 문서의 네임스페이스 관리</span><span class="sxs-lookup"><span data-stu-id="09cec-104">Managing Namespaces in an XML Document</span></span>

<span data-ttu-id="09cec-105">XML 네임스페이스는 XML 문서의 요소 및 특성 이름을 사용자 지정 및 미리 정의된 URI와 연결합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-105">XML namespaces associate element and attribute names in an XML document with custom and predefined URIs.</span></span> <span data-ttu-id="09cec-106">이러한 연결을 만들려면 네임스페이스 URI의 접두사를 정의하고 해당 접두사를 사용하여 XML 데이터의 요소 및 특성 이름을 한정합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-106">To create these associations, you define prefixes for namespace URIs, and use those prefixes to qualify element and attribute names in XML data.</span></span> <span data-ttu-id="09cec-107">네임스페이스는 요소 및 특성 이름이 충돌하는 것을 막고 동일한 이름의 요소 및 특성이 처리 및 확인되도록 하는 역할을 합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-107">Namespaces prevent element and attribute name collisions, and enable elements and attributes of the same name to be handled and validated differently.</span></span>

## <a name="declaring-namespaces"></a><span data-ttu-id="09cec-108">네임스페이스 선언</span><span class="sxs-lookup"><span data-stu-id="09cec-108">Declaring namespaces</span></span>  

 <span data-ttu-id="09cec-109">요소의 네임스페이스를 선언하려면 `xmlns:` 특성을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-109">To declare a namespace on an element, you use the `xmlns:` attribute:</span></span>  
  
 `xmlns:<name>=<"uri">`  
  
 <span data-ttu-id="09cec-110">`<name>`이 네임스페이스 접두사이고 `<"uri">`는 네임스페이스를 식별하는 URI입니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-110">where `<name>` is the namespace prefix and `<"uri">` is the URI that identifies the namespace.</span></span> <span data-ttu-id="09cec-111">접두사를 선언한 후에는 접두사를 사용하여 XML 문서의 요소 및 특성을 한정하고 네임스페이스 URI와 연결할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-111">After you declare the prefix, you can use it to qualify elements and attributes in an XML document and associate them with the namespace URI.</span></span> <span data-ttu-id="09cec-112">네임스페이스 접두사는 문서 전체에서 사용되므로 길이가 짧아야 합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-112">Because the namespace prefix is used throughout a document, it should be short in length.</span></span>  
  
 <span data-ttu-id="09cec-113">이 예제에서는 다음 두 개의 `BOOK` 요소를 정의합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-113">This example defines two `BOOK` elements.</span></span> <span data-ttu-id="09cec-114">첫 번째 요소는 접두사 `mybook`에 의해 한정되고 두 번째 요소는 접두사 `bb`에 의해 한정됩니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-114">The first element is qualified by the prefix, `mybook`, and the second element is qualified by the prefix, `bb`.</span></span> <span data-ttu-id="09cec-115">각 접두사는 다른 네임스페이스 URI와 연관되어 있습니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-115">Each prefix is associated with a different namespace URI:</span></span>  
  
```xml  
<mybook:BOOK xmlns:mybook="http://www.contoso.com/books.dtd">  
<bb:BOOK xmlns:bb="urn:blueyonderairlines" />
</mybook>
```  
  
 <span data-ttu-id="09cec-116">요소가 특정 네임스페이스의 일부임을 나타내기 위해 네임스페이스 접두사를 추가합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-116">To signify that an element is a part of a particular namespace, add the namespace prefix to it.</span></span> <span data-ttu-id="09cec-117">예를 들어 `Author` 요소가 `mybook` 네임스페이스에 속하는 경우에는 `<mybook:Author>`로 선언됩니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-117">For example, if a `Author` element belongs to the `mybook` namespace, it is declared as `<mybook:Author>`.</span></span>

## <a name="declaration-scope"></a><span data-ttu-id="09cec-118">선언 범위</span><span class="sxs-lookup"><span data-stu-id="09cec-118">Declaration scope</span></span>  

 <span data-ttu-id="09cec-119">네임스페이스는 선언 지점부터 선언된 요소의 끝까지 유효합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-119">A namespace is effective from its point of declaration until the end of the element it was declared in.</span></span> <span data-ttu-id="09cec-120">이 예에서 `BOOK` 요소에서 정의된 네임스페이스는 `BOOK` 요소와 같은 `Publisher` 요소 외부의 요소에는 적용되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-120">In this example, the namespace defined in the `BOOK` element doesn't apply to elements outside the `BOOK` element, such as the `Publisher` element:</span></span>  
  
```xml  
<Author>Joe Smith</Author>  
<BOOK xmlns:book="http://www.contoso.com">  
    <title>My Wonderful Day</title>  
      <price>$3.95</price>  
</BOOK>  
<Publisher>  
    <Name>MSPress</Name>  
</Publisher>  
```  
  
 <span data-ttu-id="09cec-121">네임스페이스는 사용되기 전에 선언되어야 하지만 XML 문서의 맨 처음에 나타나야 하는 것은 아닙니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-121">A namespace must be declared before it can be used, but it doesn't have to appear at the top of the XML document.</span></span>  
  
 <span data-ttu-id="09cec-122">XML 문서에서 여러 네임스페이스를 사용하는 경우 네임스페이스 하나를 기본 네임스페이스로 정의하여 더 간결한 문서를 만들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-122">When you use multiple namespaces in an XML document, you can define one namespace as the default namespace to create a cleaner looking document.</span></span> <span data-ttu-id="09cec-123">기본 네임스페이스는 루트 요소에서 정의되고 문서에서 정규화되지 않은 모든 요소에 적용됩니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-123">The default namespace is declared in the root element and applies to all unqualified elements in the document.</span></span> <span data-ttu-id="09cec-124">기본 네임스페이스는 요소에만 적용되고 특성에는 적용되지 않습니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-124">Default namespaces apply to elements only, not to attributes.</span></span>  
  
 <span data-ttu-id="09cec-125">기본 네임스페이스를 사용하려면 요소의 선언에서 접두사와 콜론을 생략합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-125">To use the default namespace, omit the prefix and the colon from the declaration on the element:</span></span>  
  
```xml  
<BOOK xmlns="http://www.contoso.com/books.dtd">  
...
</BOOK>
```

 <span data-ttu-id="09cec-127"><xref:System.Xml.XmlNamespaceManager> 클래스는 네임스페이스 URI 및 해당 접두사의 컬렉션을 저장하고 이 컬렉션에서 네임스페이스를 조회, 추가 및 제거할 수 있도록 합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-127">The <xref:System.Xml.XmlNamespaceManager> class stores a collection of namespace URIs and their prefixes, and lets you look up, add, and remove namespaces from this collection.</span></span> <span data-ttu-id="09cec-128">특정 컨텍스트에서 향상된 XML 처리 성능을 위해 이 클래스가 필요합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-128">In certain contexts, this class is required for better XML processing performance.</span></span> <span data-ttu-id="09cec-129">예를 들어, XPath 지원을 위해 <xref:System.Xml.Xsl.XsltContext> 클래스에서 <xref:System.Xml.XmlNamespaceManager>를 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-129">For example, the <xref:System.Xml.Xsl.XsltContext> class uses <xref:System.Xml.XmlNamespaceManager> for XPath support.</span></span>  
  
 <span data-ttu-id="09cec-130">네임스페이스 관리자는 네임스페이스에 대한 유효성 검사를 수행하지 않지만 접두사 및 네임스페이스가 이미 확인되었고 [W3C 네임스페이스](https://www.w3.org/TR/REC-xml-names/) 사양을 따르는 것으로 간주합니다.</span><span class="sxs-lookup"><span data-stu-id="09cec-130">The namespace manager doesn't perform any validation on the namespaces, but assumes that prefixes and namespaces have already been verified and conform to the [W3C Namespaces](https://www.w3.org/TR/REC-xml-names/) specification.</span></span>
