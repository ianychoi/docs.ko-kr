<span data-ttu-id="cd42b-298">가비지 수집기는 사용자 애플리케이션에서 만들어지는 대부분의 개체에 대해 메모리 관리 작업을 자동으로 수행할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="cd42b-298">For most of the objects that your application creates, you can rely on garbage collection to automatically perform the necessary memory management tasks.</span></span> <span data-ttu-id="cd42b-299">하지만 관리되지 않는 리소스의 경우는 명시적으로 정리할 필요가 있습니다.</span><span class="sxs-lookup"><span data-stu-id="cd42b-299">However, unmanaged resources require explicit cleanup.</span></span> <span data-ttu-id="cd42b-300">가장 일반적인 형태의 관리되지 않는 리소스로는 파일 핸들, 창 핸들 또는 네트워크 연결 등의 운영 체제 리소스를 래핑하는 개체를 들 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="cd42b-300">The most common type of unmanaged resource is an object that wraps an operating system resource, such as a file handle, window handle, or network connection.</span></span> <span data-ttu-id="cd42b-301">가비지 수집기에서는 관리되지 않는 리소스를 캡슐화하는 데 사용되는 관리되는 개체의 수명을 추적할 수 있지만, 리소스 정리 방법에 대한 구체적인 정보는 알 수 없습니다.</span><span class="sxs-lookup"><span data-stu-id="cd42b-301">Although the garbage collector is able to track the lifetime of a managed object that encapsulates an unmanaged resource, it doesn't have specific knowledge about how to clean up the resource.</span></span>
